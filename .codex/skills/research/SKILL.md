# 基本面/行业研究（证据导向）

## 输入 / 输出约定

- 默认输出：`outputs/agents/research.md`
- 如需新闻线索：先抓取并落盘到 `outputs/agents/news_raw.json` + `outputs/agents/news_digest.md`，再做归纳分析（线索≠事实）。

## 输出原则（别写成作文）

- 先给结论，再给证据，再写不确定性。
- 证据分级：官方/论文/专利/产品手册 > 一手访谈/会议纪要 > 媒体稿（媒体只能当线索）。
- 讲工程/商业约束：成本、可靠性、供货、验证进度、客户是谁。

## 工作流

### 1) 把问题拆成可验证子问题

例如：
- “技术路径是什么”= 关键模块/关键步骤/关键风险点
- “应用进度”= 量产与否、在轨/在车/在厂验证证据、客户/订单证据

### 2) 先定范围与时效

- 时间范围：默认近 12~24 个月；历史背景单独一节。
- 地域范围：默认全球；如果只关心国内要写清楚。

### 3) 证据检索与记录

每条证据要记录：
- 一句话结论（它证明了什么）
- 来源链接
- 可信度（高/中/低）
- 可能的反证/缺口

### 4) 输出报告

按 `references/report_template.md` 输出到 `outputs/agents/research.md`（或用户指定路径）。

## 新闻抓取（可选：你要就做，但别当圣旨）

建议两段式（防止一上来就瞎解读）：
1) 抓取标题/时间/来源/链接/摘要
2) 再做“证据清单 + 风险点/共识/分歧 + 待核验清单”

本 skill 自带一个 MVP 抓取脚本（基于东方财富搜索，支持股票代码或关键词）：

```bash
".venv/bin/python" .codex/skills/research/scripts/news_digest.py \
  --query 600641 \
  --pages 3 --page-size 10 \
  --out-json outputs/agents/news_raw.json \
  --out-md outputs/agents/news_digest.md
```

## 禁止事项（别越界）

- 媒体稿只能当线索；关键结论必须回到公告/财报/监管披露核验。
- 不编数据；缺口就写“缺失/不确定/待核验”。
- 不输出投资建议与收益承诺，只做“事实-证据-不确定性”。
